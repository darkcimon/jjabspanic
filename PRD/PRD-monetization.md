# PRD-monetization — 짭스패닉 수익화 전략

> 작성일: 2026-03-17
> 배포 타겟: 웹 앱 (모바일 브라우저, PWA)

---

## 1. 수익화 기본 원칙

- **코어 루프는 무료** — 게임 자체(선 그리기, 몬스터 피하기)는 결제 없이 끝까지 플레이 가능
- **콘텐츠로 수익화** — 결제 동기는 "더 좋은 그림을 빠르게 보고 싶다"는 욕구
- **강제 광고 최소화** — 광고는 선택형(보상형)만 허용, 강제 삽입은 이탈률 상승 유발

---

## 2. 수익 모델 구성

### 2-1. 보상형 광고 (Google AdSense / Kakao AdFit)

| 트리거 | 보상 |
|---|---|
| 스테이지 실패 후 광고 시청 | 현재 스테이지 이어하기 1회 |
| 광고 시청 | 추가 목숨 1개 지급 |
| 광고 시청 | 다음 스테이지 캐릭터 미리보기 (블러 해제 5초) |

광고 시청은 **유저가 직접 선택**. 스킵 가능하도록 구성.

### 2-2. 일회성 결제 — 콘텐츠 팩 (핵심 수익원)

| 상품 | 가격 | 내용 |
|---|---|---|
| 갤러리 팩 A (스테이지 1~100) | 3,900원 | 해당 구간 캐릭터 이미지 전체 즉시 해금 |
| 갤러리 팩 B (스테이지 101~200) | 3,900원 | 동일 |
| 갤러리 팩 C (스테이지 201~300) | 3,900원 | 동일 |
| 완전판 팩 (전 구간) | 6,900원 | 팩 A+B+C 통합 할인 |

이미 플레이로 해금된 스테이지는 팩 구매 시 중복 지급 없이 잔여분만 해금.

**로그인 없이 구매 가능** — 결제 완료 시 서버에서 구매 코드 발급, 사용자가 코드를 보관하면 기기 변경 시도 복구 가능.

### 2-3. 커스텀 보상 이미지 생성 (선택 기능)

- 유저가 키워드를 입력해 자신만의 캐릭터 이미지를 생성하는 기능 (`/api/reward/generate`)
- 기본 무료 1회 제공, 이후 **300원/회**
- Stability AI API 실비용 대비 마진 확보 가능 (SD3: 약 $0.065/장 ≈ 95원)

## 3. 수익화 퍼널

```
게임 플레이
    │
    ├─ 스테이지 클리어 → 캐릭터 일부 공개 → "전체 보기" 버튼 → 팩 구매 유도
    │
    ├─ 스테이지 실패 → "이어하기? 광고 보기" → 광고 수익
    │
    └─ 갤러리 화면 → 잠긴 이미지 미리보기 → 팩 구매 유도
```

## 4. 웹 앱 배포 시 결제 구현 방향

### 4-1. 결제 연동

- **토스페이먼츠 위젯** 을 `web/js/payment.js`에 구현
- 로그인 없이 일반 카드 결제 → 결제 완료 시 서버에서 **구매 코드** 발급
- 구매 코드는 `server/data/purchases.json`에 저장 (imageStore.js와 동일한 패턴)
- 사용자가 코드를 보관하면 기기 변경 시 `/payment/redeem` 엔드포인트로 복구

### 4-2. 광고 연동

- **Google AdSense** (웹 기반) 또는 **Kakao AdFit** 적용
- 보상형 광고는 현재 AdSense에서 직접 지원하지 않으므로, `web/js/ad.js`에 광고 팝업 → 타이머 완료 → 보상 지급 플로우를 직접 구현
- 또는 **Google Ad Manager**의 rewarded ad unit 사용

### 4-3. PWA 전환 (앱스토어 없이 "앱처럼" 배포)

- `manifest.json` + Service Worker 추가로 PWA 변환
- 안드로이드에서는 "홈 화면에 추가" 후 전체화면 실행
- **TWA(Trusted Web Activity)** 로 Google Play 스토어 등록 가능 (웹뷰 기반)
  - 스토어 등록 시 결제를 Google Play 인앱결제로 교체해야 할 수 있음 (정책 확인 필요)

## 5. 단계별 수익화 로드맵

| 단계 | 목표 | 주요 작업 |
|---|---|---|
| Phase 1 | 광고 수익 검증 | AdSense/AdFit 연동, 보상형 광고 플로우 구현 |
| Phase 2 | 콘텐츠 팩 판매 | 토스페이먼츠 일회성 결제, 구매 코드 발급, purchases.json 저장 |
| Phase 3 | 스토어 배포 | PWA 변환, TWA로 Play Store 배포 |
| (선택) | 구독 도입 | 트래픽 확보 후 필요 시 — 카카오 로그인 + 월정액 결제 추가 |

## 6. 수익 예측 (보수적 추정)

| 지표 | 가정 |
|---|---|
| MAU | 1,000명 (Phase 2 기준) |
| 광고 수익 | 1,000 MAU × 3회/월 × 100원 = **300,000원/월** |
| 콘텐츠 팩 | 20건/월 × 3,900원 = **78,000원/월** |
| **합계** | **약 378,000원/월** |

> 실제 수익은 DAU, 광고 단가(CPM), 팩 구매율에 따라 크게 달라집니다.
